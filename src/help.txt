Usage: `%PROGRAM_NAME% <verb> [switches]...` where <verb> is one of the following:
    `encrypt`: Encrypts and authenticates the incoming data-stream and writes it to stdout; supported switches:
        `--password=<string>` (OPTIONAL):
            The password to encrypt/decrypt the data; you must either specify a password using this switch or set it
            using the environment-variable "CRYPTO_STREAM_PASSWORD"

        `--password=<string>` (OPTIONAL):
            The password to encrypt/decrypt the data; you must either specify a password using this switch or set it
            using the environment-variable "CRYPTO_STREAM_PASSWORD"

        `--pbkdf-iterations=<integer>` (DEFAULT: 8):
            The amount of KDF-iterations required to derive the key from the password

        `--pbkdf-memory-requirements=<integer>` (DEFAULT: 256MiB):
            The amount of memory required to derive the key from the password in KiB

        `--pbkdf-parallelism=<integer>` (DEFAULT: 4):
            The amount of threads required to derive the key from the password (WARNING: different values will produce
            different keys)

    `decrypt`: Verifies and decrypts the incoming data-stream and writes it to stdout; supported switches:
        `--password=<string>` (OPTIONAL):
            The password to encrypt/decrypt the data; you must either specify a password using this switch or set it
            using the environment-variable "CRYPTO_STREAM_PASSWORD"

        `--password=<string>` (OPTIONAL):
            The password to encrypt/decrypt the data; you must either specify a password using this switch or set it
            using the environment-variable "CRYPTO_STREAM_PASSWORD"

        `--pbkdf-iterations=<integer>` (DEFAULT: 8):
            The amount of KDF-iterations required to derive the key from the password

        `--pbkdf-memory-requirements=<integer>` (DEFAULT: 256MiB):
            The amount of memory required to derive the key from the password in KiB

        `--pbkdf-parallelism=<integer>` (DEFAULT: 4):
            The amount of threads required to derive the key from the password (WARNING: different values will produce
            different keys)

    `licenses`: Display the license-information

    `help`: Display this help
